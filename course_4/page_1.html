Основы работы с Cisco IOS

Задача

В данном сценарии мы разберемся как подключаться к сетевому оборудованию и изучим основы работы с операционной системой Cisco IOS.

Цель работы

В данном блоке мы научимся:

  • -подключаться к оборудованию по консоли
  • -задавать пароль для привелегированного режима enable
  • -создавать пользователей
  • -выбирать тип удалённого подключения
  • -устанавливать и включать авторизацию на подключение к консоли для удалённых пользователей

Доступные команды

Мы хотим увидеть список всех доступных команд в этом режиме.

Введите команду, которая используется для просмотра всех доступных команд:

Switch >?

Клавишу Enter нажимать не надо.

Доступные команды в режиме enable

Теперь войдите в привилегированный режим.

Switch >enable

Просмотрите список доступных команд в привилегированном режиме. Как видите их гораздо больше, потому как привелегированный режим предоставляет гораздо больше возможностей.

Switch #?

Работа с Cisco IOS

Выйти из привилегированного режима можно при помощи команды Disable или exit. Для автоматического дописания команд можно использовать табулцию.

Для примера напишите

Switch> en

И нажмите Tab. Тут же появится

Switch> enable

Однако если при наличии вариативности команд, Tab не поможет.

Switch> e?

Вы увидели список доступных команд.

Установка имени устройства

Перейдём в режим конфигурации

Switch #config terminal

Switch (config)#

Имя хоста сетевого устройства используется для локальной идентификации.

Когда вы входите в сетевое устройство, вы видите имя хоста перед символом режима (">" или "#"). Это имя может быть использовано для определения места нахождения.

Установите "Switch1" как имя вашего сетевого устройства.

Switch (config)#hostname Switch 1

Switch1(config)#

Пароль доступа

Пароль доступа позволяет вам контролировать доступ в привилегированный режим. Это очень важный пароль, потому что в привилегированном режиме можно вносить конфигурационные изменения. Установите пароль доступу "password1".

Switch1(config)#enable password password1

Теперь испытаем этот пароль. Выйдите из сетевого устройства и попытайтесь зайти в привилегированный режим.

Введите пароль и если всё правильно вы войдёте в привилегированный режим.

Пароль доступа

Пароль

Введите пароль и если всё правильно вы войдёте в привилегированный режим.

Однако это не самый надёжный способ защиты, потому как любой, получивший прямой доступ к коммутатору в привилегированном может узнать пароль при помощи команды

Switch1# show run

Пароль доступа

Теперь пароль выглядит по другому

Для решения данной проблемы можно использовать шифрование пароля

Switch1(config)# service password-encryption

Проверим

Switch1(config)# exit

Switch1# show run

Теперь ваш пароль зашифрован, и злоумышленник не сможет так просто заполучить его

Пароль доступа

Если вы введёте старый пароль

С вас потребуют новый

Так же можно использовать параметр secret

Switch1# conf t

Switch1(config)# enable secret password2

Switch1(config)# exit

Switch1# show run

Этот пароль автоматически зашифрован и теперь приоритетен. Без него войти в привилегированный режим невозможно.

Локальный пользователь

Теперь создадим локального пользователя. Это можно сделать в режиме конфигурации терминала. Не забывайте пользоваться «?» и «Tab» для упрощения

Switch1(config)# username admin privilege 15 password adminpass

Мы задали имя пользователя admin, задали ему самую высокую, пятнадцатую привилегию и пароль adminpass.

Чтобы включить авторизацию пользователя набираем

Switch1(config)# line console 0

Switch1(config-line)# login local

Выходим из привилегированного режима

Локальный пользователь

Теперь для входа нужна авторизация

Создание пользователя и настройка авторизации завершины, пора двигаться дальше.

Настройка адреса

Перейдём к настройке ip адреса.

В коммутаторе ip адреса всегда настраиваются на логических интерфейсах. Для этого заходим в режим конфигурации терминала.

Switch1(config)# interface Vlan1

Switch1(config-if)# ip address 192.168.0.1 255.255.255.0

Switch1(config-if)# no shutdown

Ip адрес настроен. Теперь настроим виртуальные терминальные линии. Для этого входим в режим конфигурирования терминальных линий.

Switch1(config)# line vty 0 4

Настройка виртуальных терминальных линий

Здесь мы можем выбрать транспортный протокол, а именно входящий telnet, потому как в Cisco Packet Tracer отсутствует SSH клиент.

Switch1(config-line)# transport input telnet

Switch1(config-line)# login local

Выходим из режима конфигурирования

Switch1(config-line)#exit

Switch1(config)#exit

Теперь необходимо сохранить изменения

Switch1# wr mem

Проверим правильность конфигурации

Соединяем PC с Switch кабелем типа Copper Straight-Throught.

Устанавливаем PC ip адрес 192.168.0.2 и маску подсети 255.255.255.0.

Удалёный доступ

Удалённый доступ

Открываем его командную строку и проверяем ping с адресом коммутатора 192.168.0.1.

C:\> ping 192.168.0.1

Первый запрос возможно будет потерян, но это нормально, если остальные три пройдут успешно.

Теперь проверим telnet.

C:\> telnet 192.168.0.1

На этом всё

Если все предыдущие действия выполнены правильно, мы получим удалённый доступ к терминалу, и у вас должен быть запрошен логин и пароль для входа в терминал.

Таким образом мы завершили настройки, которые как правило производятся при подключении настоящего коммутатора, но через программу Cisco Packet Tracer.
